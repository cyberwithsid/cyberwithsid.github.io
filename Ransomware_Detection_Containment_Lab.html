Simulated LockBit 3.0, Conti, and BlackCat (ALPHV) ransomware variants in an isolated lab environment, built behavioral detection rules based on observed TTPs, and validated IR containment playbooks against live ransomware execution. No production systems were involved.

LockBit Encryption Detection Rule
yaml — Sigma Rule 
# Detect ransomware encryption: mass file rename + extension change
title: Ransomware Mass File Rename
mitre_attack: [T1486]
detection:
  selection:
    EventID: 4663  # Object access - file rename
    ObjectName|endswith:
      - .lockbit
      - .conti
      - .alphv
  condition: selection | count() > 50 | within 60s
level: critical

Containment Automation

powershell — Auto-Containment Script 
# Ransomware auto-containment — triggered by SIEM alert
param([string]$HostName, [string]$IncidentID)

# 1. Immediately isolate the host at firewall level
netsh advfirewall set allprofiles firewallpolicy blockinbound,blockoutbound

# 2. Disable infected user account
Disable-ADAccount -Identity (Get-WmiObject Win32_ComputerSystem).UserName

# 3. Kill suspicious processes matching ransomware patterns
Get-Process | Where-Object {$_.CPU -gt 80 -and $_.Name -notmatch "svchost|csrss|lsass"} |
  Stop-Process -Force

# 4. Write containment event to log
Write-EventLog -LogName Application -Source "IR-AutoContain" \
  -EventId 9001 -Message "Host $HostName isolated for $IncidentID"

Designed a Zero Trust Network Architecture (ZTNA) for a hybrid cloud environment hosting 2,500+ users across on-premises and Azure cloud workloads. The architecture replaces the legacy castle-and-moat perimeter model with identity-based, continuous verification across every access request.

The design follows NIST SP 800-207 (Zero Trust Architecture) and CISA's Zero Trust Maturity Model, with phased implementation covering identity verification, device trust, network microsegmentation, and application-layer access controls.

// 02

Zero Trust Principles

Never Trust, Always Verify

Every user, device, and network flow must be authenticated and authorized — regardless of network location.

Least Privilege Access

Grant minimum necessary permissions, just-in-time, with continuous validation and automatic expiration.

Assume Breach

Design controls assuming adversaries are already inside. Limit blast radius through microsegmentation.

Continuous Monitoring

Log all access attempts, validate device health at every request, and detect anomalies in real time.

    "users": { "includeUsers": ["All"] },
    "applications": { "includeApplications": ["All"] }
  },
  "grantControls": {
    "operator": "AND",

Network Microsegmentation

bash — Azure NSG Microsegmentation 
# Create NSG for production workload tier — deny all by default
az network nsg create --name NSG-ProdTier --resource-group RG-ZeroTrust

# Allow ONLY necessary inbound from Zscaler ZPA broker IPs
az network nsg rule create \
  --nsg-name NSG-ProdTier \
  --name Allow-ZPA-Broker \
  --priority 100 \
  --source-address-prefixes 165.225.0.0/17 \
  --destination-port-ranges 443 \
  --access Allow

# Deny everything else inbound
az network nsg rule create \
  --nsg-name NSG-ProdTier --name DenyAllInbound \
  --priority 4096 --access Deny --direction Inbound

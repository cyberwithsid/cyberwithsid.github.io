Conducted a PCI DSS v4.0 gap assessment for a national retail organization processing over 2 million card transactions annually. The assessment evaluated compliance against all 12 PCI DSS requirements and identified 34 control gaps — each with a prioritized remediation roadmap aligned to the March 2025 transition deadline.

PCI DSS v4.0 introduced significant changes from v3.2.1, including a customized approach option, new multi-factor authentication requirements, and enhanced anti-phishing controls. This assessment specifically focused on transition-gap items new to v4.0.

Critical Gap Details

Req 8 — MFA Gaps (Critical)
  • MFA not enforced for all non-console access to CDE systems
  • Shared service accounts in use for card processing components
  • Default vendor passwords not changed on 3 network devices
  • MFA enforced for admin console access — compliant
Req 3 — Account Data Protection Gaps
  • PANs stored unencrypted in 2 legacy log files — CRITICAL
  • Tokenization implemented for main transaction flow — compliant
  • Key custodian documentation incomplete — needs formal split knowledge procedure
CRIT

Unencrypted PAN storage (Req 3.4) is a Critical finding. Immediate containment required: restrict log access, initiate data discovery scan, and engage encryption team within 48 hours.

Key Lessons for v4.0 Transition

  • Start the v4.0 transition assessment 12 months before the compliance deadline
  • v4.0 Req 6.4.3 (script management) and Req 11.6.1 (change detection) are new and often overlooked
  • Customized Approach option requires detailed documentation and Board-level sign-off — plan early
  • Automated log review (Req 10.7.2) needs SIEM tuning — manual review will not satisfy QSA
  • PAN discovery scans revealed 3 additional systems storing card data — always scan broadly

The Wazuh Manager is the brain of the operation — it receives agent data, decodes events, evaluates detection rules, and triggers alerts. The OpenSearch Indexer stores all logs for retention and search, while the Dashboard presents a unified view for SOC analysts.

Note

For environments with over 500 endpoints, consider deploying the Wazuh Indexer on dedicated hardware separate from the Manager to avoid resource contention during peak ingestion windows.

OS Hardening

Before deploying Wazuh, harden the underlying OS. Apply updates, configure the firewall, enable auditing, and disable unnecessary services. This reduces the attack surface of your SOC infrastructure itself.

Warn

Run these commands on each server node (Manager, Indexer, Dashboard) before installation. Ensure UFW rules align with your network topology — overly restrictive rules will break inter-component communication.

bash — OS Hardening 
# Update and upgrade packages
apt update && apt upgrade -y

# Install security essentials
apt install -y fail2ban ufw auditd

# Configure firewall — allow required ports only
ufw allow 22,443,1514,1515,9200
ufw enable

# Enable and start auditd for system call auditing
systemctl enable --now auditd

# Disable root SSH login
sed -i 's/PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
systemctl restart sshd
Kernel Hardening (sysctl)
bash — sysctl tuning 
# Append hardening params to sysctl.conf
cat >> /etc/sysctl.conf <<EOF
net.ipv4.ip_forward = 0
net.ipv4.conf.all.rp_filter = 1
net.ipv4.conf.all.accept_redirects = 0
net.ipv4.tcp_syncookies = 1
kernel.randomize_va_space = 2
EOF
sysctl -p

All-in-One Installation

Wazuh provides an automated installation script that deploys the Manager, Indexer, and Dashboard on a single node. This is ideal for lab environments, small deployments, or initial PoC setups.

Tip

The all-in-one installer handles certificate generation, service configuration, and inter-component connectivity automatically. For distributed deployments, use the -d flag with individual component installers.

bash — Download & Install 
# Download the official Wazuh install script
curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh

# Run all-in-one deployment (-a flag)
bash wazuh-install.sh -a

# Verify all services are running post-install
systemctl status wazuh-manager
systemctl status wazuh-indexer
systemctl status wazuh-dashboard
Post-Install Validation
bash — Health Check 
# Check cluster health via OpenSearch API
curl -k -u admin:<password> https://localhost:9200/_cluster/health?pretty

# Confirm Wazuh Manager is receiving events
tail -f /var/ossec/logs/alerts/alerts.log

Agent Deployment

Wazuh agents are lightweight daemons deployed on endpoints that collect system logs, file integrity events, network traffic summaries, and vulnerability data — forwarding everything to the Manager for analysis.

Windows Agent — MSI Deployment
cmd — Windows 
:: Silent MSI install pointing to Wazuh Manager IP
msiexec /i wazuh-agent.msi /qn WAZUH_MANAGER='10.0.0.10'

:: Start the agent service
NET START WazuhSvc
Linux Agent — DEB / RPM
bash — Ubuntu / Debian 
# Download Wazuh agent package
curl -so wazuh-agent.deb \
  https://packages.wazuh.com/4.x/apt/pool/main/w/wazuh-agent/wazuh-agent_4.x.deb

# Install the package
WAZUH_MANAGER='10.0.0.10' dpkg -i wazuh-agent.deb

# Enable and start the agent
systemctl daemon-reload
systemctl enable wazuh-agent
systemctl start wazuh-agent
Agent Group Management
bash — Manager CLI 
# List all connected agents
/var/ossec/bin/agent_control -lc

# Create agent group for Windows endpoints
curl -k -u wazuh-wui:<pwd> -X POST \
  "https://localhost:55000/groups" \
  -H "Content-Type: application/json" \
  -d '{"group_id":"windows-endpoints"}'

Final Notes

This deployment provides a complete enterprise-grade SOC using Wazuh SIEM with full detection, automation, and compliance readiness. The combination of real-time alerting, MITRE ATT&CK mapping, file integrity monitoring, and compliance dashboards makes Wazuh a powerful open-source alternative to expensive commercial SIEM platforms.

Key next steps after base deployment include tuning detection rules to reduce false positives, integrating a SOAR platform (such as TheHive or Shuffle) for automated response, and connecting external threat intelligence feeds via MISP integration to enrich alerts with contextual IOC data.

Developed and deployed automated CIS (Center for Internet Security) Benchmark hardening scripts for RHEL 9, Ubuntu 22.04, Windows Server 2022, and AWS CIS Foundations. The automation uses Ansible playbooks and OpenSCAP for assessment, achieving an average compliance score of 92% across all targets.

All hardening is idempotent — scripts can be re-run at any time and will only change settings that have drifted from the benchmark. Continuous compliance monitoring was implemented via scheduled OpenSCAP scans with results forwarded to the SIEM.

// 02

Linux Hardening — Ubuntu/RHEL

CIS Level 1 – Filesystem Configuration
bash — Partition Hardening 
# Ensure /tmp is on a separate partition with noexec
echo "tmpfs /tmp tmpfs defaults,rw,nosuid,nodev,noexec,relatime 0 0" >> /etc/fstab
mount -a

# Disable unused filesystems (CIS 1.1.1)
for fs in cramfs freevxfs jffs2 hfs hfsplus udf; do
  echo "install $fs /bin/true" >> /etc/modprobe.d/CIS.conf
done
SSH Hardening (CIS 5.2)
bash — SSH Config 
# Apply CIS SSH hardening settings
cat >> /etc/ssh/sshd_config << EOF
Protocol 2
PermitRootLogin no
PasswordAuthentication no
PermitEmptyPasswords no
MaxAuthTries 4
ClientAliveInterval 300
ClientAliveCountMax 0
LoginGraceTime 60
AllowTcpForwarding no
X11Forwarding no
EOF
systemctl restart sshd

Windows Server Hardening

CIS Level 1 – Account Policies
powershell — Account Policy 
# Set password policy per CIS Windows Server 2022 L1
net accounts /minpwlen:14 /maxpwage:60 /minpwage:1 /uniquepw:24

# Enable audit policies
auditpol /set /subcategory:"Logon" /success:enable /failure:enable
auditpol /set /subcategory:"Account Logon" /success:enable /failure:enable
auditpol /set /subcategory:"Privilege Use" /success:enable /failure:enable

# Disable SMBv1 (CIS 18.3.3)
Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force

Ansible Automation

yaml — Ansible Playbook 
# cis-ubuntu-22.yml — CIS Ubuntu 22.04 L1 Hardening
---
- name: CIS Ubuntu 22.04 Level 1 Hardening
  hosts: ubuntu_servers
  become: yes
  vars:
    cis_level: 1
  roles:
    - cis-ubuntu-22

# Run with:
ansible-playbook -i inventory/production cis-ubuntu-22.yml --check # Dry run
ansible-playbook -i inventory/production cis-ubuntu-22.yml           # Apply
// 05

OpenSCAP Compliance Scanning

bash — OpenSCAP Assessment 
# Install OpenSCAP
apt install -y openscap-scanner scap-security-guide

# Run CIS Level 1 assessment against Ubuntu 22.04
oscap xccdf eval \
  --profile xccdf_org.ssgproject.content_profile_cis_level1_server \
  --results /tmp/cis-scan-results.xml \
  --report  /tmp/cis-scan-report.html \
  /usr/share/xml/scap/ssg/content/ssg-ubuntu2204-ds.xml

# Forward score to SIEM via syslog
logger -t CIS_SCAN "$(oscap info --fetch-remote-resources /tmp/cis-scan-results.xml | grep Score)"

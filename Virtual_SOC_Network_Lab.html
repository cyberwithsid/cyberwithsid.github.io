A homelab environment built to simulate a corporate network with realistic attack paths, SIEM log collection, IDS/IPS, and Blue Team practice exercises. The lab enables hands-on practice in threat hunting, log analysis, malware analysis, and incident response without risk to production systems.

The network topology mimics a small enterprise: AD domain, DMZ, workstations, a web server, and an attacker machine. All traffic is captured and forwarded to the SIEM for analysis.

SIEM Stack Setup

bash — ELK + Wazuh 
# Install Elasticsearch
apt install -y elasticsearch
systemctl enable --now elasticsearch

# Install Logstash with Wazuh pipeline
apt install -y logstash
curl -so /etc/logstash/conf.d/wazuh.conf \
  https://packages.wazuh.com/integrations/logstash/wazuh-logstash.conf

# Install Kibana and Wazuh plugin
apt install -y kibana
bin/kibana-plugin install https://packages.wazuh.com/wazuhapp/wazuhapp.zip

Practice Exercises

Exercise 1 – Brute Force Detection

Run Hydra from Kali against SSH on the Ubuntu Web Server. Observe how Wazuh triggers a brute force alert after 5 failed attempts, and practice isolating the source IP via pfSense.

bash — Hydra SSH Brute Force 
# From Kali — attack SSH on web server
hydra -l root -P /usr/share/wordlists/rockyou.txt \
  10.10.10.30 ssh -t 4
Exercise 2 – Mimikatz Credential Dump

Execute Mimikatz on WIN10-01 and observe EDR and Wazuh alerts for LSASS memory access, correlating with Windows Event ID 4656 (credential access).
